Bridge Model

Iteration with automated visualizations

Notes
Ideally, let's pretend we have a visualization in an automated tool, that I'm just going to represent with this abstract circle - We'd want to be able to open it in a drawing program - make some changes in the drawing program - and move it back, with the changes, into the automated program - where we could make more changes - and so on and so forth. Unfortunately, we can't do this right now; current automated visualization tools don't let you edit existing graphics.
Part of the reason is that our picture is a little simplistic; from an authoring perspective, a visualization is more than just graphics. - Each modality comes with extra, non-transferable elements that only exist in one tool. - For example, on the drawing side, a tool might provide color swatches, layers, or typography support that you can't get anywhere else - An automated tool has formalized rules that make it possible to manipulate graphics in a scalable way, as well as unique features of their own, such as force-directed layouts
So how do we make this work when the features that we need are spread across multiple programs? The core idea of the bridge model is to make changes in parallel, and merge their effects on the graphics. So to repeat the earlier workflow, we start with an automated visualization, - copy it into the drawing program - make drawing changes - make automated changes - merge the results - and bring the merged result back into each tool
In practice, it might look something like this. Let's pretend that I'm creating that HBO infographic from earlier. - I might start by generating the layout with D3 - and then exporting that to Illustrator, - where I change the colors and typography - (zoom in) Now if we zoom in, we can see that the actors are sorted first name... what if I wanted to sort by last name? - (zoom out) - (merge) I would merge my color changes back over to D3 - (reintegrate) (zoom in) - where I can change how they're sorted - (zoom out) Now let's imagine that our client comes to us, and they want the graphic to fit in a square instead of this letter-sized page. - (merge) If we merge back to the Illustrator, - (reintegrate) it's easy to crop off the bottom of the page... but we're also going to need a new layout because the bottom part of our graphic is going to get cut off - (clean) This time, let's make the change on both sides in parallel. - So we can crop the page on the right using Illustrator - We can implement a new layout on the d3 side, - and merge the changes! - Let's reintegrate the results... - (clean) And now that title is kind of in the wrong spot... - So let's move it with Illustrator - (merge) And merge that back over to d3 - (reintegrate)
